# 評分流程說明

## 一、手動觸發：「立即計算今日評分」完整流程

當使用者在前端按下「立即計算今日評分」按鈕，系統會依序執行以下步驟：

```
使用者按下按鈕
      │
      ▼
前端呼叫 POST /api/v1/signals/run-now
      │
      ▼
後端讀取要計算的股票清單
（有設定股票池 → 用股票池；股票池是空的 → 用預設5檔）
      │
      ▼
對每一檔股票執行評分計算（含 AI 分析）
      │
      ├── 1. 查詢法人買賣超（呼叫證交所 TWSE API）
      ├── 2. 查詢股價、均線、成交量（呼叫 yfinance）
      ├── 3. 查詢殖利率（呼叫 yfinance）
      ├── 4. 查詢台指走勢（呼叫 yfinance ^TWII）
      └── 5. 綜合計算總分 → 呼叫 Claude AI 分析
      │
      ▼
將結果寫入資料庫（signals 表）
（若該股票今日已有記錄且在 1 小時內更新過，則跳過不重算）
      │
      ▼
回傳所有股票的評分結果給前端
```

## 二、每個分數項目的計算說明

### 分項 1：法人買賣超得分（institutional_score）

**白話說明**：外資和投信今天是在買還是在賣這檔股票？

**資料來源**：台灣證券交易所（TWSE）官網的法人買賣超報表

**計算規則**：
```
外資買賣超股數 + 投信買賣超股數 = 合計淨買超

合計淨買超 > 1,000 張  →  +2 分（大量買入，正面訊號）
合計淨買超 > 0 張      →  +1 分（小量買入，偏正面）
合計淨買超 ≤ 0 張      →  -2 分（賣超，負面訊號）
查不到資料             →   0 分
```

**分數範圍**：-2 到 +2

---

### 分項 2：均線得分（ma_score）

**白話說明**：今天的股價和過去 20 天的平均價格比較，是貴還是便宜？

**資料來源**：yfinance（從 Yahoo Finance 抓近 30 天的收盤價）

**計算規則**：
```
今日收盤價 < 20日均線 × 0.98  →  +2 分（跌破均線 2%，可能是買點）
今日收盤價 ≤ 20日均線 × 1.05  →  +1 分（在均線附近，價格合理）
今日收盤價 > 20日均線 × 1.05  →  -1 分（漲太多超過均線 5%，偏貴）
```

**分數範圍**：-1 到 +2

---

### 分項 3：成交量得分（volume_score）

**白話說明**：今天的成交量和最近 5 天的平均比，是熱絡還是冷清？

**資料來源**：yfinance（近 30 天的日成交量）

**計算規則**：
```
今日成交量 / 近 5 日平均量 = 量比

量比 > 1.5   →  +2 分（成交量明顯放大，市場活躍）
量比 0.7~1.5 →   0 分（成交量正常）
量比 < 0.7   →  -1 分（成交量萎縮，市場冷清）
```

**分數範圍**：-1 到 +2

---

### 分項 4：殖利率得分（yield_score）

**白話說明**：這檔股票的現金股利殖利率高不高？殖利率越高代表配息越豐厚。

**資料來源**：yfinance（從 Yahoo Finance 取得殖利率資訊）

**計算規則**：
```
殖利率 ≥ 6.5%  →  +2 分（非常高殖利率）
殖利率 5.0~6.5% →  +1 分（不錯的殖利率）
殖利率 < 5.0%  →  -1 分（殖利率偏低）
查不到資料      →   0 分
```

**分數範圍**：-1 到 +2

---

### 分項 5：期貨/市場情緒得分（futures_score）

**白話說明**：今天台灣股市大盤的整體氣氛是好是壞？上漲的市場裡個股比較容易跟漲。

**資料來源**：yfinance（台股加權指數 ^TWII）

**計算規則**（三個小項目加總，最後限制在 -3 到 +3）：

```
① 台股指數漲跌幅
   今日漲幅 > +1%  →  +1 分（大盤上漲）
   今日跌幅 < -1%  →  -1 分（大盤下跌）
   漲跌幅在 ±1% 內 →   0 分（橫盤）

② 成交量配合度
   量增且上漲  →  +1 分（有量有價，趨勢確立）
   量增且下跌  →  -1 分（有量下跌，賣壓重）
   其他情況    →   0 分

③ 期現貨價差（目前固定為 0，此功能預留）
   價差 > 50 點  →  +1 分
   價差 < -50 點 →  -1 分
   其他          →   0 分

最後：分數限制在 [-3, +3] 之間
```

**分數範圍**：-3 到 +3

---

### 總分計算

```
總分 = 法人分 + 均線分 + 成交量分 + 殖利率分 + 期貨分

理論最高：2 + 2 + 2 + 2 + 3 = +11 分
理論最低：-2 + (-1) + (-1) + (-1) + (-3) = -8 分
```

---

### AI 分析（claude-sonnet-4-6 模型）

**觸發條件**：`run-now`（手動觸發）或 排程任務中 **總分 ≥ 6 分**的前 10 檔

**分析內容**：把五個分項的分數和股票資訊傳給 Claude，由 AI 綜合判斷

**輸出格式**：
- `ai_action`：buy（建議買進）/ watch（建議觀察）/ sell（建議賣出）
- `ai_reason`：50 字以內的白話說明（繁體中文）

**容錯處理**：若 AI 服務故障，自動降級為 `watch`（觀察），並填入預設說明

## 三、進入「今日推薦」的條件

今日推薦（`GET /api/v1/signals/top`）要同時滿足以下三個條件才會出現：

1. **日期是今天**：必須是今天的評分結果
2. **總分 ≥ 6 分**：評分達到門檻（預設）
3. **目前沒有持倉**：模擬交易中這檔股票沒有 status=open 的未平倉部位

> 說明：第三點是為了避免重複買進同一檔股票，若已有持倉就不再出現在推薦清單。

## 四、排程任務（14:00 自動執行）的完整流程

每個工作日（週一到週五）下午 2 點，系統自動執行以下流程：

```
下午 14:00（台灣時間）觸發
        │
        ▼
─── 第一階段：純數字評分 ───────────────────────────────────
        │
        ▼
從資料庫讀取股票池清單
（若股票池為空，改用預設 5 檔：0050、0056、2886、2412、5880）
        │
        ▼
對每一檔股票執行評分計算（跳過 AI，只算數字分數）
  ↳ 法人分 + 均線分 + 成交量分 + 殖利率分 + 期貨分 = 總分
        │
        ▼
將結果寫入資料庫（有的更新、沒有的新增）
        │
        ▼
─── 第二階段：AI 分析 ──────────────────────────────────────
        │
        ▼
找出今日總分 ≥ 6 分的前 10 檔股票
        │
        ▼
對這些股票呼叫 Claude AI 分析
（每檔股票單獨分析，依序執行）
        │
        ▼
將 AI 建議（ai_action、ai_reason）更新回資料庫
        │
        ▼
─── 第三階段：模擬持倉自動平倉 ─────────────────────────────
        │
        ▼
查詢所有 status=open 的模擬持倉
        │
        ▼
取得最新股價，計算損益
  ↳ 損益 ≥ +6%  →  停利平倉（賣出）
  ↳ 損益 ≤ -3%  →  停損平倉（賣出）
  ↳ 其他        →  繼續持倉
        │
        ▼
更新 simulation_trades 表（status 改為 closed，記錄損益）
        │
        ▼
排程任務完成
```

## 五、每週股票池篩選流程（週一 08:00）

```
週一 08:00（台灣時間）觸發
        │
        ▼
呼叫 TWSE API 取得所有上市公司清單（約 900+ 檔）
        │
        ▼
對每一檔進行逐一驗證（透過 yfinance）
  ↳ 殖利率 ≥ 4%？（太低的不要）
  ↳ 市值 ≥ 50 億？（太小的不要）
  ↳ 排除特定產業？（醫療健保、半導體等波動大的產業）
        │
        ▼
通過篩選的股票清單寫入 stock_pool 表
（先清空舊資料，再寫入新資料）
        │
        ▼
更新 filter_status 表，狀態改為 completed
```

## 六、防呆機制

| 情況 | 系統的處理方式 |
|------|------------|
| 1 小時內已算過評分 | 跳過，直接回傳舊結果（避免重複浪費資源） |
| 全量評分任務已在跑 | 拒絕新的請求，回傳錯誤訊息 |
| yfinance 取不到股價 | 沿用上次的均線/成交量/殖利率分數（fallback 機制） |
| AI 服務連線失敗 | 自動填入 watch + 預設說明，不影響其他功能 |
| 期貨資料取不到 | futures_score 設為 0，不影響其他分項 |
